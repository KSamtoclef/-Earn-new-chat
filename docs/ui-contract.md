# Earn Chat UI preservation contract

## Must remain recognizable

- Dark green/black Earn Chat palette and bright teal/green accents.
- Current mobile-first spacing, rounded cards, badges, progress visuals, and button hierarchy.
- Landing, registration, login, dashboard, chat, share, claim, leaderboard, profile, withdrawal, processing, and admin concepts.
- Five-day progress explanation and prominent balance/progress presentation.

## Allowed advancement

- Safer forms, accessible labels, focus states, and readable error messages.
- Stable mobile viewport and safe-area behavior.
- Coherent structured conversations and contextual suggestions.
- Large inline sponsored cards within chat.
- Server-authoritative wallet, task, and journey state.
- Faster rendering, fewer network requests, and lazy admin pages.
- Clear privacy and verification wording.

## Prohibited change

- Generic SaaS redesign or ChatEarn branding.
- Floating reward popups or competing sponsored modals.
- Client-calculated money or client-selected reward amounts.
- Versioned, hotfix, patch, final, or stabilizer runtime assets.
- Critical progress stored only in local storage.
- Random conversation replies unrelated to the latest message.

Every rebuilt screen requires a mobile side-by-side comparison with the archived source before acceptance.
