# Delivery roadmap

1. **Foundation** — repository, archived source, architecture, UI contract, and compatibility plan.
2. **Read-only Supabase audit** — inventory and reconciliation report.
3. **Canonical database** — schema, RLS, indexes, private writers, public RPCs, verification, and rollback.
4. **Application shell** — non-versioned CSS/JS modules with visual parity.
5. **Core journey** — auth, dashboard, coherent chat, rewards, tasks, and sharing.
6. **Withdrawal journey** — KYC, withdrawal, processing, resume, and admin review.
7. **Admin and operations** — lazy pagination, offers, users, performance, errors, and audit.
8. **QA and cutover** — concurrency, mobile, returning users, security, staging, reconciliation, and domain cutover.

Only one delivery step is active at a time. No module is marked complete until its blocking verification passes.
